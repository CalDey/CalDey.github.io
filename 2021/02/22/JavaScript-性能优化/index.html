JavaScript-性能优化

发表于 2021-02-22 更新于 2021-03-04 分类于 JavaScript

阅读全文，了解更多

1.性能优化原则

多使用内存，缓存或其他方法

减少CPU计算量，减少网络加载耗时

(所有编程的性能优化 -> 空间换时间)

2.性能优化角度

2.1.让加载更快

减少资源体积：压缩代码/图片/视频
减少访问次数：合并代码（雪碧图合并图片），SSR服务器端渲染，缓存
使用CDN

2.2.让渲染更快

CSS放在head，JS放在body后
用DOMContentLoaded触发
懒加载（图片懒加载，上滑显示更多）
缓存DOM查询
频繁DOM操作合并到一起插入DOM(document.createDocumentFragment())
节流（throttle）和防抖（debounce）

3.性能优化案例

3.1.资源合并

3.2.缓存

webpack中给静态资源加hash后缀，根据文件内容计算hash值，文件不变hash不变

url和文件不变，返回304，自动触发http缓存机制

3.3.CDN

静态资源加速

3.4.SSR

服务器端渲染：将网页和数据一起加载一起渲染

前后端分离：先加载网页，再加载数据，再渲染数据

3.5.图片懒加载

先加载预览图，后加载真实图片

3.6.DOM性能优化

缓存DOM查询
将频繁操作改为一次操作
